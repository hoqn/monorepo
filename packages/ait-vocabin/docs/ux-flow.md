# 독일어 단어 학습 앱 — UX 플로우 설계서

> 작성일: 2026-03-21
> 상태: 초안 (기획 확정 전)

---

## 1. 개요

앱인토스 WebView 미니앱 (React Web). 독일어 명사(성/복수형) 학습을 Spaced Repetition 기반 퀴즈로 제공하며, 인터랙션과 UX 자체로 감성을 살리는 것이 핵심 차별점. 캐릭터 마스코트 없이 CSS 애니메이션·햅틱(토스 브릿지)·피드백으로 앱의 개성을 표현한다.

---

## 2. 네비게이션 구조

탭 바 없는 단일 플로우. 화면 간 이동은 컨텍스트에 따라 자연스럽게 연결.

```
홈
├── [프로필 버튼] → 프로필 화면
├── [리더보드 미리보기 탭] → 리더보드 화면
└── [세션 시작] → 세션 화면
                    ├── 세션 진행 중
                    ├── [목숨 0] → 서브세션 진입 안내 → 복습 서브세션 → 목숨 회복 안내 → 세션 복귀
                    └── [세션 완료] → 세션 종료 화면 → 홈
```

---

## 3. 전체 사용자 플로우

### 3-1. 최초 실행 (온보딩)

```
앱 실행
→ 온보딩 1: 앱 소개 (Spaced Repetition, 목숨 시스템 등 핵심 개념)
→ 온보딩 2: 레벨 진단 테스트 (짧은 퀴즈로 초보/중급 자동 판별)
→ 온보딩 3: 학습 목표 설정 (하루 목표 세션 수)
→ 온보딩 4: 알림 설정 (시간대 선택: 새벽/아침/점심/낮/저녁/밤)
→ 홈 화면
```

### 3-2. 일반 학습 플로우

```
홈
→ 세션 시작 버튼 탭
→ 세션 화면 (10–15문항)
  → 문항 반복:
      단어 카드 표시
      → 답안 선택
      → 카드 반응 (정답: 물리적 팝 / 오답: 흔들림 + 색 변화) + 햅틱
      → 하단 피드백 패널 슬라이드업 (정답/오답 + 간단 설명)
      → 사용자가 "다음" 버튼 탭 → 다음 문항
→ 세션 종료 화면
→ 홈
```

### 3-3. 목숨 소진 → 복습 서브세션 플로우

```
세션 진행 중 목숨 0
→ [안내 모달] "오늘 조금 어려웠나요? 틀린 단어를 빠르게 복습하면 목숨이 돌아와요"
   → [복습 시작] 버튼
→ 복습 서브세션 (틀린 단어만, 단축 세션)
→ 복습 완료 → [안내 화면] "목숨 회복! 다시 도전해볼까요?"
   → [계속하기] → 원래 세션 복귀 (목숨 리셋)
```

## 4. 화면 목록

### S01 — 온보딩 (4 스텝)

| 항목 | 내용 |
|------|------|
| 목적 | 앱 개념 설명 + 초기 설정 |
| 진입 | 최초 실행 시 1회만 |
| 진출 | 홈 화면 |
| 스텝 구성 | 앱 소개 → 레벨 선택 → 목표 설정 → 알림 설정 |

**각 스텝 구성**
- 상단: 일러스트 or 애니메이션 영역
- 중단: 제목 + 설명 텍스트
- 하단: CTA 버튼 (다음 / 완료)
- 스텝 인디케이터 (점 4개)

**온보딩 2 — 레벨 진단 테스트**
- "몇 가지 단어를 보여드릴게요" 안내 후 짧은 퀴즈 (5문항 내외)
- 결과에 따라 자동으로 초보/중급 커리큘럼 배정
- 결과 화면: "초보 코스로 시작해요!" / "중급 단어도 포함해드릴게요!" 표시 후 다음 스텝으로

**온보딩 4 — 알림 시간 선택**
- 선택지: 새벽 (6시 전) / 아침 (7–9시) / 점심 (12–13시) / 낮 (15–17시) / 저녁 (18–20시) / 밤 (21–23시)
- 선택지 카드 형태로 표시, 1개 선택
- "나중에 설정" 옵션도 제공

---

### S02 — 홈 화면

| 항목 | 내용 |
|------|------|
| 목적 | 학습 현황 확인 + 빠른 세션 진입 |
| 진입 | 최초 로딩, 세션 종료 후 |
| 진출 | 세션 화면, 리더보드, 프로필 |

**레이아웃 구성 (위 → 아래)**

```
┌─────────────────────────────┐
│  앱 로고/이름       [프로필] │  ← 토스 스타일 프로필 버튼 (우상단)
├─────────────────────────────┤
│  🔥 12일 스트릭              │
│  Lv.7  ████████░░  420 XP  │  ← 레벨 + XP 진행 바
├─────────────────────────────┤
│                             │
│   [ 세션 시작하기 ]          │  ← 메인 CTA (크고 눈에 띄게)
│   복습 예정 단어 8개 포함    │  ← Spaced Repetition 어필
│                             │
├─────────────────────────────┤
│  이번 주 리더보드            │
│  1위 홍길동   1,240 XP      │
│  2위 김철수   980 XP        │
│  3위 나       420 XP   [→]  │  ← 탭하면 전체 리더보드로
├─────────────────────────────┤
│  오늘 학습: 24단어           │  ← 간단한 오늘 통계
└─────────────────────────────┘
```

**설계 의도**
- 스트릭과 XP는 매일 앱을 여는 동기 부여
- "복습 예정 단어 N개 포함"으로 Spaced Repetition을 서비스 어필 포인트로 활용
- 리더보드 미리보기는 홈 내 스크롤 섹션 → 탭 시 전체 리더보드로 이동

### S03 — 세션 화면

| 항목 | 내용 |
|------|------|
| 목적 | 핵심 학습 경험 |
| 진입 | 홈 → 세션 시작, 서브세션 완료 후 복귀 |
| 진출 | 세션 종료 화면, 서브세션 진입 안내 |

**레이아웃**

```
┌─────────────────────────────┐
│ [X]  ████████████░░  ❤❤❤❤❤ │  ← 닫기 + 프로그레스 바 + 목숨
├─────────────────────────────┤
│                             │
│         Hund                │  ← 단어 카드
│          개                 │  ← 한국어 뜻
│                             │
├─────────────────────────────┤
│  이 단어의 성(관사)은?       │  ← 문제 유형 안내
│                             │
│  [ der ]  [ die ]  [ das ]  │  ← 성 퀴즈: 가로 3버튼
│                             │  ← 복수형 퀴즈: 세로 리스트
└─────────────────────────────┘
```

**정답/오답 피드백 (조합 방식)**
- 1단계 — 카드 즉각 반응:
  - 정답: 카드 위로 팝업 + 초록 글로우 + 성공 햅틱
  - 오답: 좌우 흔들림 + 빨간 테두리 + 에러 햅틱
- 2단계 — 하단 피드백 패널 슬라이드업:
  - 정답: 초록 배경 + "정답!" + 간단 설명 (예: "der Hund — 남성 명사") + "다음" 버튼
  - 오답: 빨간 배경 + "오답" + 정답 표시 + 설명 + "다음" 버튼
- 사용자가 "다음" 버튼을 탭해야 다음 문항으로 넘어감

### S04 — 서브세션 진입 안내

| 항목 | 내용 |
|------|------|
| 목적 | 목숨 소진 → 복습 전환 설명 |
| 진입 | 세션 중 목숨 0 |
| 진출 | 복습 서브세션 |

**구성**
- 일러스트 영역 (아쉬움/격려 표현)
- 제목: "조금 어려웠나요?"
- 설명: "틀린 단어를 빠르게 복습하면 목숨이 돌아와요"
- CTA: "복습 시작하기"
- 선택: "그냥 끝내기" (세션 종료로 이동)

### S05 — 복습 서브세션

S03 세션 화면과 동일한 레이아웃. 단, 상단에 "복습 중" 뱃지 표시. 틀린 단어만 출제.

### S06 — 목숨 회복 안내

| 항목 | 내용 |
|------|------|
| 목적 | 복습 완료 → 세션 복귀 동기 부여 |
| 진입 | 복습 서브세션 완료 |
| 진출 | 원래 세션 복귀 |

**구성**
- 일러스트/애니메이션 (목숨 회복 표현)
- 제목: "목숨 회복!"
- 설명: "다시 도전해볼까요?"
- CTA: "계속하기" → 원래 세션 복귀 (목숨 리셋)

### S07 — 세션 종료 화면

| 항목 | 내용 |
|------|------|
| 목적 | 세션 결과 요약 + 동기 부여 |
| 진입 | 세션 전체 완료 |
| 진출 | 홈 |

**구성**
```
┌─────────────────────────────┐
│       세션 완료!             │
│   [광고 영역 or 일러스트]    │  ← 광고 불가 시 일러스트 fallback
├─────────────────────────────┤
│  정답률    정답  획득 XP     │
│   82%     10/12   +85      │
│  새로 배운 단어: 5개         │
├─────────────────────────────┤
│        [ 닫기 ]             │  ← 홈으로
└─────────────────────────────┘
```

## 5. 인터랙션 & 상태 설계

### 답안 카드 버튼 상태

| 상태 | 시각 | 햅틱 |
|------|------|------|
| 기본 | 중간 톤 카드 | — |
| 눌림(active) | 살짝 스케일 다운 | light |
| 정답 | 초록 배경, 위로 팝 | success (묵직) |
| 오답 | 빨간 배경, 흔들림 | error (짧고 날카롭게) |
| 비활성(다른 답 선택 후) | 흐리게 fade | — |

### 피드백 패널

- 하단에서 슬라이드업 (spring 애니메이션)
- 정답: 초록 테마
- 오답: 빨간 테마 + 정답 강조 표시

### 세션 → 서브세션 전환

- 목숨 아이콘이 하나씩 깨지는 애니메이션
- 마지막 목숨 소진 시 화면 전체 살짝 흔들림 + 강한 햅틱
- 이후 안내 모달 슬라이드업

## 6. 디자인 시스템 방향 (TDS 기반)

- **기반**: TDS (`@toss/tds-mobile`)
- **컬러**: TDS 기본 팔레트 위에 학습 앱 시맨틱 컬러 추가
  - 정답: Green (TDS success 계열)
  - 오답: Red (TDS error 계열)
  - 목숨: 따뜻한 Red/Pink 계열
  - 스트릭: Orange/Amber 계열
- **타이포**: TDS 기본 타이포그래피
  - 단어 카드 단어: 크고 굵게 (Display 수준)
  - 뜻: 보조 텍스트 (Body)
- **스페이싱**: 8px 베이스 그리드
- **터치 타겟**: 최소 44pt (iOS HIG 기준)
- **접근성**: WCAG AA 이상, 충분한 컬러 대비
